# Комплексное отслеживание событий на сайте

### Решаемая проблема

Серверная интеграция с Roistat была неработоспособна на протяжении длительного времени, критически важные данные о поведении пользователей не попадали в CRM.

### Решение

Разработка клиентской интеграции, которая перехватывает 7 ключевых событий на сайте и передает их в Roistat.

Что делает:
1. Собирает параметры, связанные с событием (например, имя товара, ID модального окна, GET-параметры).
2. Извлекает Google Client ID из cookies для сопоставления пользователя в Roistat.
3. Структурирует данные (корзина, формы) перед отправкой.

### Подход

* **События:** использует обработчики событий для отслеживания действий пользователя.
* **Асинхронность:** использует асинхронную логику (Promises) для проверки загрузки DOM, готовности модулей Roistat и диагностики блокировки запросов браузером.
* **Самодиагностика:** выводит в консоль цветные, структурированные сообщения об ошибках и этапах работы в реальном времени.

### Инструменты

* JavaScript
* Creatium API
* Roistat API
