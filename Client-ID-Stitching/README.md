# Client ID Stitching: сквозная идентификация пользователя

### Решаемая проблема

Стандартные формы захвата лидов не передают критически важные идентификаторы, такие как Google Client ID (_ga) и Yandex Metrika User ID (_ym_uid). Это приводит к "разрыву" воронки: невозможно связать действия пользователя на сайте с его последующим лидом в CRM.

### Решение

Скрипт выполняет Client ID Stitching для устранения разрыва данных. Он извлекает и форматирует идентификаторы _ga и _ym_uid из cookies, а перед отправкой формы автоматически добавляет эти идентификаторы в скрытые поля.

### Результат

Внешние аналитические системы и CRM получают полный набор данных, что позволяет клиенту точно определять, какой маркетинговый канал привел к конверсии.

### Подход

* Чистый парсинг: для извлечения и форматирования ID используются нативные методы JS, что исключает зависимость от сторонних библиотек.
* Устойчивость к изменениям в DOM: привязка к событию "before-submit" на уровне API платформы гарантирует, что инъекция данных произойдет до отправки, независимо от разметки формы.
